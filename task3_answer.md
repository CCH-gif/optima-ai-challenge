## 2. 租户水管漏水投诉自动处理流程

### 完整链路设计

### 详细步骤说明

| 步骤 | 负责人/系统 | 操作内容 |
|------|-------------|----------|
| 1. 邮件接收 | 邮件监听服务 | 通过 IMAP 定时拉取邮件 |
| 2. 内容解析 | NLP 模型 | 提取：姓名、房号、问题描述、附件图片 |
| 3. 意图识别 | 分类模型 | 判断是否为报修请求，提取关键信息（漏水位置、严重程度） |
| 4. 工单创建 | 工单系统 | 生成工单，关联租户信息、房屋信息 |
| 5. 优先级判定 | 规则引擎 | 根据关键词（"严重"、"漏水量大"）判定紧急程度 |
| 6. 自动回复 | 邮件服务 | 发送确认邮件，告知预计处理时间 |
| 7. 通知相关人 | 消息推送 | 通过微信/短信通知物业经理和维修班组 |
| 8. 派单分配 | 调度系统 | 根据维修班组位置、空闲情况自动派单 |
| 9. 维修处理 | 维修人员 | 现场维修，上传维修前后对比照片 |
| 10. 租户确认 | 消息推送 | 发送维修完成通知，请求确认 |
| 11. 满意度回访 | 问卷系统 | 自动发送满意度调查 |
| 12. 归档结算 | 工单系统 | 工单关闭，数据统计归档 |

### 关键技术点

- **邮件解析**：使用正则 + NLP 提取关键信息
- **图像识别**：如果附件包含漏水照片，用 CV 模型评估严重程度
- **定时任务**：每分钟拉取一次新邮件
- **状态机**：工单状态流转（待处理→处理中→已完成→已关闭）

## 3. Git rebase vs merge 区别

### 核心区别

| 特性 | merge | rebase |
|------|-------|-------|
| **原理** | 合并两个分支的更改，创建一个 merge commit | 将分支的基址移到另一个分支的顶端 |
| **历史** | 保留完整分叉历史，清晰但可能杂乱 | 产生线性历史，更干净 |
| **风险** | 较低，适合协作分支 | 较高，会改写历史，不能对公共分支使用 |
| **适用** | 合并功能分支到主分支 | 整理本地提交，使提交历史简洁 |

### 我的使用习惯

- **merge**：合并功能分支到 main/dev 分支，保留完整历史
- **rebase**：
  - 整理本地未推送的 commits（squash、fixup）
  - 将 feature 分支基于最新的 main 分支更新
  - pull --rebase 拉取远程更新时保持线性历史

### 黄金法则

> **永远不要对已推送到远程的 commits 进行 rebase**

## 4. 接到陌生技术需求的第一步

**第一步：明确需求边界和背景**

具体行动：

1. **提问确认**：
   - 这个需求的业务目标是什么？
   - 谁是最终用户？
   - 有没有类似的现有功能可以参考？
   - 优先级和截止时间？

2. **技术调研**：
   - 查找相关文档、代码
   - 了解现有系统架构
   - 确定技术可行性

3. **评估复杂度**：
   - 拆解为子任务
   - 识别依赖项和风险点

**核心思路**：在动手之前，先确保理解"做什么"和"为什么做"，避免做无用功。

## 5. 对「做了但没记录 = 没做」的理解

### 我的解读

这句话强调**可追溯性**和**团队协作**：

| 维度 | 含义 |
|------|------|
| **信息透明** | 别人不知道你做了什么，就无法复用你的成果 |
| **知识传承** | 文档是团队资产，没记录=资产流失 |
| **责任认定** | 无记录=无凭证，出现问题时难以追溯 |
| **协作效率** | 好的记录让团队成员快速上手，减少重复沟通 |

### 实践做法

- 写好 commit message（描述做了什么、为什么）
- 及时更新技术文档
- 在任务管理工具中记录进度
- 代码 review 是另一种形式的"记录"

**一句话总结**：记录是为了让工作成果可复用、可追溯、可传承。

## 6. 3小时完成任务后做什么

### 我的做法

1. **自我检查**
   - 是否有遗漏的边界情况？
   - 代码是否经过测试？
   - 是否有 lint/typecheck 问题？

2. **主动找事**
   - 回顾本周目标，看有没有可以提前完成的
   - 主动帮助同事 code review
   - 优化现有代码/文档
   - 学习新技术（但不影响正常工作）

3. **提前规划**
   - 规划明天的工作
   - 整理工作笔记

### 核心心态

不浪费时间，但也不"假装忙碌"。高效完成后，与其等任务不如主动找价值。
